## 픽이 에이전트에게 실어 보내는 것

element 를 픽하면 위젯은 스크린샷이 아니라 **에이전트가 추론할 근거**를 함께 실어 보냅니다:

<div class="adt-evidence not-content">
  <img
    src="/picker-evidence.png"
    alt="agent-devtools composer 에 픽한 element 의 근거 tooltip — 소스 파일, 컴포넌트 체인, 고유 CSS selector 가 표시된 모습."
  />
  <ul>
    <li>
      <strong>컴포넌트 체인</strong> — 루트까지의 named 조상, 예:{' '}
      <code>OrderSummary → App</code>.
    </li>
    <li>
      <strong>소스 파일</strong> — element 가 렌더되는 실제 파일. 에이전트가
      올바른 코드를 먼저 엽니다.
    </li>
    <li>
      <strong>Selector + outerHTML</strong> — 정확한 노드. 소스 경로가 안 잡혀도
      에이전트가 마크업을 찾을 수 있습니다.
    </li>
  </ul>
</div>
